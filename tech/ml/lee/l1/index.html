李宏毅机器学习（Lecture 1）

视频：YouTube 课程

A. 本讲核心结论

  • 机器学习可以概括为：让机器自动找到一个合适的函数（function）。
  • 三大任务：回归（regression）、分类（classification）、结构化学习（structured learning）。
  • 经典三步：模型（model） → 损失（loss） → 优化（optimization）。
  • 案例中模型能改进，靠的是领域知识（domain knowledge）（例如发现每周周期后再改输入特征）。

B. 什么是机器学习

这节课最重要的一句话：机器学习就是让机器具备「找函数」的能力。
给定输入，机器输出你想要的结果；难点在于这个输入到输出的映射往往极其复杂，人手写不出显式公式，因此要用数据和算法把函数「学」出来。

课堂例子

  • 语音识别：声音讯号 → 文字
  • 影像辨识：图片 → 内容标签
  • AlphaGo：棋盘状态 → 下一步落子位置

C. 三大任务

1）回归（regression）

输出是一个标量（scalar），即连续数值。
例子：预测明天中午的 PM2.5 数值。

2）分类（classification）

输出是从预先给定的类别（classes）里选一个，像做选择题。
例子：

  • 垃圾邮件：是 / 否（yes / no）
  • AlphaGo 也可看成多类分类：在 19×19 个可落子位置中选一个

3）结构化学习（structured learning）

输出不是单个数字或单个类别，而是有结构的对象（如整篇文章、整张图、复杂序列等）。
例子：机器写文章、画图等。

D. 三步骤：模型 → 损失 → 优化

第一步：写出带未知参数的函数（模型 model）

先假设函数形式，其中含有待学习的参数。例如预测隔天观看人次，可先写成：

y = b + w · x₁
  • x₁：特征（feature），已知输入（如前一天总观看人次）
  • w：权重（weight），与特征相乘的参数
  • b：偏置（bias），加在后面的常数项

要点：这种「先猜函数长什么样」往往来自你对问题的领域知识（domain knowledge），而不是凭空拍脑袋。

第二步：定义损失函数（loss）

损失（loss）本身也是一个函数：输入是模型参数（如 w、b），输出表示「这组参数好不好」。
在训练数据上，把预测值 y 与 标签（label） 即真实值 ŷ（y-hat） 的差距逐日算出来再平均，就得到总损失 L。

常见差距形式：

  • 平均绝对误差（MAE, mean absolute error）：对每个样本算「预测与真实的差距」取绝对值，再取平均
  • 均方误差（MSE, mean square error）：对每个样本算「预测与真实之差」再平方，再取平均
  • 交叉熵（cross-entropy）：多用于输出是概率分布的分类（classification）问题

把不同 (w, b) 对应的损失画成等高线图，称为误差曲面（error surface）。

注意：损失函数的形式是你自己定义的；因此一般讨论里损失曲线形状可以很灵活。若明确定义成「绝对值误差」这类非负量，则不会出现负的损失值。

第三步：优化参数（optimization）

目标：找到使损失 L 最小的参数，记为 w*、b*。
本课使用梯度下降（gradient descent）。

流程图（梯度下降在做什么）

flowchart TD
    A["随机初始化 w0, b0"] --> B["计算梯度 dL/dw, dL/db"]
    B --> C["按学习率 eta 更新"]
    C --> D{"满足停止条件?"}
    D -- 否 --> B
    D -- 是 --> E["得到 w*, b*"]
  1. 随机给定初始 w₀、b₀
  2. 计算损失对参数的梯度（gradient）（即误差曲面在该点的斜率方向）
  3. 沿「让损失下降」的方向更新参数；步长由学习率（learning rate） η（eta）控制——η 是超参数（hyperparameter），需你自己设定
  4. 重复直到「迭代次数上限」或「梯度接近 0」等停止条件

公式

w ← w − η · (∂L / ∂w)

b ← b − η · (∂L / ∂b)

直觉：若在某一位置 ∂L/∂w < 0，说明往 w 增大的方向走损失会下降，应把 w 增大；若 ∂L/∂w > 0，则应把 w 减小。
两个参数时，(Δw, Δb) 的合成方向就是梯度（gradient）的反方向，在二维误差曲面上沿「下山」方向走一步。

局部极小与全局最小（local minima / global minima）
课上用一维曲线说明：梯度为 0 的点可能是局部极小（local minima），不一定是全局最小（global minima）。老师提到：深度学习中「局部极小」常被过度渲染，真正难点后续课再讲；此处先建立几何直觉即可。

E. 课程案例：YouTube 频道观看人次预测

任务：输入频道后台的历史统计（如过去每日观看），输出隔天总观看人次。

版本 1：只用前一天

模型：y = b + w·x₁（x₁ 为前一天观看）。

  • 在 2017–2020 训练数据上，平均误差约 0.48k
  • 在 2021 未见过数据上，平均误差约 0.58k

利用领域知识改进：看出「每周周期」

真实曲线有明显周期性（例如周五、周六偏低）。仅用「前一天」相当于把红线大致右移一天，预测形状缺乏周期结构。

改为把前 7 天的观看都作为特征（各天乘以不同权重 wⱼ 再与 b 相加，课堂上写成对 j=1…7 求和的形式）。

  • 训练误差约 0.38k，测试约 0.49k（比只看 1 天好）

再扩大窗口：

  • 28 天：训练约 0.33k，测试约 0.46k
  • 56 天：训练约 0.32k，测试仍约 0.46k（再加长收益有限）

结论：更多历史天数不一定一直变好，存在收益递减；改进往往来自对数据的理解（领域知识），而不只是堆参数。

上述「特征线性加权再偏置」的形式，统称为线性模型（linear model）。

G. 复盘问题
